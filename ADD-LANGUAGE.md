# Як додати нову мову

У застосунку два способи додати мову. **Спосіб Б (JSON)** — без перекомпіляції, підходить навіть для мов/діалектів без системної культури. **Спосіб А (.resx)** — «вбудована» мова, компілюється в застосунок.

Усі текстові ключі беруться з `FamilyTree.App/Resources/Strings.resx` (українська — базова мова). Відсутні в перекладі ключі автоматично підставляються з української, тож застосунок не «ламається» на неповному перекладі.

---

## Спосіб Б — JSON-файл (швидко, без збірки)

Цей спосіб реалізує клас `CustomLanguageFile`; файли підхоплюються при старті з теки:

```
%AppData%\FamilyTree\languages\<код>.json
```

де `<код>` — ідентифікатор мови (ім'я файлу без розширення), напр. `pl`, `de`, `uk-poltava`, `elvish`.

Кроки:

1. Створи теку `%AppData%\FamilyTree\languages\`, якщо її немає.
2. Додай файл `<код>.json` такого вигляду:

```json
{
  "displayName": "Polski",
  "formattingCulture": "pl",
  "strings": {
    "MainWindow_Title": "Drzewo genealogiczne",
    "Menu_File": "Plik",
    "Menu_New": "Nowy",
    "Menu_Open": "Otwórz…",
    "Tab_Tree": "Drzewo",
    "Tab_WhoIsWho": "Kto kim jest"
  }
}
```

- `displayName` — назва в перемикачі мов (якщо пропустити — покажеться код).
- `formattingCulture` — культура для дат/чисел (напр. `pl`, `en`). Якщо порожня чи невідома — береться запасна.
- `strings` — пари «ключ ресурсу → переклад». Перелік ключів дивись у `Strings.resx`. Достатньо перекласти потрібні; решта підставиться з української.

3. Запусти застосунок — мова з'явиться у перемикачі.

> Порада: щоб мати повний перелік ключів, візьми всі `name="…"` зі `Strings.resx` і заповни значення своєю мовою. Битий JSON тихо ігнорується (не ламає старт), тож перевір валідність файлу.

---

## Спосіб А — вбудована мова (.resx, потребує збірки)

Підходить для мов із реальною `CultureInfo` (напр. `pl`, `de`, `fr`).

1. Створи сателітний ресурс `FamilyTree.App/Resources/Strings.<код>.resx` (напр. `Strings.pl.resx`), скопіювавши всі ключі зі `Strings.resx` і переклавши значення.
2. Зареєструй мову в `FamilyTree.App/Localization/LocalizationService.cs` — у конструкторі, поряд з наявними:

```csharp
RegisterBuiltIn("uk");
RegisterBuiltIn("en");
RegisterBuiltIn("pl");   // ← нова мова
```

3. Перебудуй проєкт (`dotnet build`). Мова з'явиться у перемикачі; назва підтягнеться з `CultureInfo.NativeName`.

---

## Важливо: назви родинних зв'язків

Пункти вище перекладають **інтерфейс**. А ось **назви спорідненості** (бейджі на дереві, «Хто кому») генеруються кодом окремими форматерами, а не з resx/JSON.

Диспетчер `FamilyTree.App/Kinship/CultureKinshipFormatter.cs` зараз обирає: `en` → англійський форматер, будь-яка інша мова → **український**. Тобто нова мова отримає перекладений інтерфейс, але родинні назви будуть українською (як відкат).

Щоб повноцінно локалізувати й назви спорідненості:

1. Додай реалізацію `IKinshipFormatter` для своєї мови в `FamilyTree.Domain/Kinship/` (за зразком `UkrainianKinshipFormatter` / `EnglishKinshipFormatter`; алгоритм спільний — форматер лише перетворює `KinshipContext` на рядок).
2. Підключи її у `CultureKinshipFormatter` (додай гілку за кодом мови й форвардни `Style`).
3. Додай тести у `FamilyTree.Tests/Domain/` за зразком наявних форматерів.

---

## Коротко

- Треба лише перекласти інтерфейс, швидко й без збірки → **JSON** у `%AppData%\FamilyTree\languages\`.
- Треба «вшиту» мову в застосунок → **.resx** + `RegisterBuiltIn`.
- Треба ще й правильні назви родичів → додатковий **`IKinshipFormatter`** + гілка у `CultureKinshipFormatter`.
