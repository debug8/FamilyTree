# Додавання мов та діалектів

Застосунок підтримує два способи додати мову інтерфейсу.

## 1. Вбудована мова (є системна CultureInfo)

Для «звичайної» мови з наявною `CultureInfo` (напр. польська `pl`, німецька `de`):

1. Додати файл ресурсів `FamilyTree.App/Resources/Strings.<culture>.resx` (напр. `Strings.pl.resx`) з тими самими ключами, що й у `Strings.resx`.
2. Зареєструвати код у `LocalizationService` (виклик `RegisterBuiltIn("pl")`).
3. Для назв родинних зв'язків — реалізувати відповідний `IKinshipFormatter` (Етап 3).

Форматування дат/чисел для таких мов бере реальну `CultureInfo`.

## 2. Додаткова мова або діалект без CultureInfo (файл перекладу)

Для мов, яких немає в системі (рідкісні діалекти, штучні мови, регіональні варіанти),
переклад кладеться **файлом**, без будь-якої `CultureInfo`.

Розташування:

```
%AppData%\FamilyTree\languages\<code>.json
```

де `<code>` — ідентифікатор мови (ім'я файлу без розширення), напр. `uk-poltava.json` або `elvish.json`.
Цей `<code>` зберігається в `settings.json` і показується як обрана мова.

### Формат файлу

```json
{
  "displayName": "Полтавська говірка",
  "formattingCulture": "uk",
  "strings": {
    "MainWindow_Title": "Родинне дерево",
    "MainWindow_Welcome": "Вітаємо у Родинному дереві",
    "Settings_Language": "Мова",
    "StatusBar_Today": "Сьодні",
    "StatusBar_NoFile": "Файл не відкрито",
    "StatusBar_PersonsCount": "Осіб: {0}"
  }
}
```

Поля:

- `displayName` — назва в перемикачі мов. Якщо порожнє — береться `<code>`.
- `formattingCulture` — **необов'язково**. Код реальної культури для форматування дат/чисел
  (напр. `"uk"`, `"en"`). Якщо порожнє чи невідоме — застосовується запасна культура
  (українська за замовчуванням). Саме це дозволяє мати мову зовсім без CultureInfo.
- `strings` — пари «ключ ресурсу → переклад». **Відсутні ключі** автоматично беруться з
  нейтральної (української) мови, тож перекладати все одразу не обов'язково.

### Правила

- Файли підхоплюються під час **запуску** застосунку. Після додавання файлу — перезапустити застосунок.
- Якщо `<code>` збігається з вбудованою мовою (`uk`, `en`) — файл ігнорується (вбудована має пріоритет).
- Битий JSON не ламає запуск — такий файл просто пропускається.
